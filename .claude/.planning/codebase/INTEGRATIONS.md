## APIs & External Services

**Google Fonts (CDN):**
- Service: Google Fonts API — loads Inter and JetBrains Mono typefaces
- Integration point: `frontend/index.html` via `<link href="https://fonts.googleapis.com/css2?...">` tag
- Auth: None (public CDN)

**Internal Prediction API:**
- Service: Vercel Python Serverless Function
- Endpoint: `POST /api/predict`
- Implementation: `frontend/api/predict.py`
- Called by: `frontend/src/pages/Predict.tsx` via `fetch("/api/predict", { method: "POST" })`
- Request shape: `{ "dataset": "creditcard"|"paysim", "features": { [name: string]: number } }`
- Response shape: `{ fraud: boolean, ensemble_score: number, scores: Record<string, number>, threshold: number, latency_ms: number }`
- Auth: None (no authentication on the endpoint)

## Data Storage

**Databases:**
- None — no database is used. All historical/analytical data is stored as pre-generated static JSON files.

**Static JSON Data Files:**
- Location: `frontend/public/data/{dataset}/` served as static assets
- Datasets: `creditcard/` and `paysim/` subdirectories, plus `combined/`
- Files per dataset:
  - `summary.json` — dataset-level statistics
  - `model_results.json` — per-model evaluation metrics
  - `confusion_matrices.json` — confusion matrix data per model
  - `roc_curves.json` — ROC curve points per model
  - `pr_curves.json` — Precision-Recall curve points per model
  - `transactions.json` — sample transaction records with fraud scores
  - `distributions.json` — amount/time distributions
  - `hyperparameters.json` — model hyperparameter records
  - `sample_transactions.json` — labelled fraud/normal scenario examples for the Predict page
- Combined files: `public/data/combined/comparison.json`, `ranking.json`, `best_models.json`
- Generated by: `scripts/generate_dashboard_data.py`
- Accessed via: `frontend/src/lib/data.ts` using `fetch()` with in-memory `Map` cache

**File Storage:**
- ML model artifacts stored in `models/` directory at project root (not committed — excluded via `.gitignore`)
- Artifact types: `scaler_{tag}.pkl`, `xgboost_{tag}.json`, `iforest_{tag}.pkl`, `autoencoder_{tag}.onnx`, `autoencoder_{tag}_config.json`, `ensemble_{tag}.json`
- At runtime on Vercel, models are resolved from `/var/task/models/` (see `frontend/api/predict.py` `_get_models_dir()`)

**Caching:**
- Client-side in-memory cache only: `frontend/src/lib/data.ts` uses a `Map<string, unknown>` to cache fetched JSON responses for the session lifetime

## ML Model Integration

**Model Loading:**
- Models are lazy-loaded and cached in a module-level `_models` dict in `frontend/api/predict.py`
- First request per dataset tag (`cc` or `ps`) triggers disk load; subsequent requests use the cache
- Model path resolution order: sibling `../models/`, grandparent `../../models/`, then `/var/task/models/`

**Ensemble Composition:**
- XGBoost (`xgboost_{tag}.json`) — supervised classifier, outputs fraud probability
- Isolation Forest (`iforest_{tag}.pkl`) — unsupervised anomaly detector, score normalized via range stored in `ensemble_{tag}.json`
- Autoencoder (`autoencoder_{tag}.onnx`) — reconstruction-error anomaly detector, run via `onnxruntime`, optional (skipped if ONNX file absent)
- Final score: weighted sum defined in `ensemble_{tag}.json` `weights` field
- Fraud threshold: stored in `ensemble_{tag}.json` `threshold` field

**Training Pipeline:**
- Source data: `creditcard.csv` and `PS_20174392719_1491204439457_log.csv` at project root (not committed)
- Training script: `scripts/train_models.py`
- Dashboard data generation: `scripts/generate_dashboard_data.py`
- Training dependencies: `scripts/requirements.txt` (numpy, pandas, scikit-learn, xgboost, tensorflow, tf2onnx, onnxruntime, joblib)
